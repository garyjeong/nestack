# Nestack 화면 설계서

## 1. 개요

### 1.1 화면 구성
- **B2C 사용자 화면**: 일반 사용자(부부/커플)를 위한 모바일 스타일 웹 애플리케이션
- **어드민 화면**: 관리자를 위한 데스크톱 중심 관리자 인터페이스

---

## 2. B2C 사용자 화면

### 2.1 디자인 원칙

#### 2.1.1 디자인 컨셉
- **모바일 스타일의 웹**: 카드 기반 레이아웃, 세로 스크롤 중심, 터치 친화적 UI
- **미니멀리즘**: 불필요한 요소 제거, 핵심 정보에 집중
- **게이미피케이션**: 미션 완료, 뱃지 획득 등 성취감을 주는 시각적 피드백
- **프라이버시 존중**: 데이터 공유 설정을 명확하게 표시

#### 2.1.2 레이아웃 원칙
- **카드 기반**: 주요 정보는 카드 형태로 구성
- **단일 컬럼**: 모바일처럼 한 줄씩 배치 (데스크톱에서도 동일)
- **최대 너비 제한**: 데스크톱에서도 콘텐츠 영역 최대 너비 제한 (예: 600px)
- **충분한 여백**: 터치 영역 최소 44px × 44px
- **고정 하단 네비게이션**: 주요 기능 접근성 향상

#### 2.1.3 색상 및 타이포그래피
- **주 색상**: (추후 결정)
- **강조 색상**: 미션 완료, 성공 상태 표시
- **텍스트**: 명확한 계층 구조, 가독성 우선
- **아이콘**: 직관적이고 일관된 아이콘 시스템

### 2.2 화면 목록

#### 2.2.1 인증 화면
- 로그인
- 회원가입
- 이메일 인증
- 비밀번호 찾기
- 비밀번호 재설정

#### 2.2.2 메인 화면
- 대시보드 (홈)

#### 2.2.3 가족 관리 화면
- 가족 그룹 생성/초대
- 초대 코드 입력
- 가족 설정
- 데이터 공유 설정

#### 2.2.4 미션 화면
- 미션 목록
- 미션 생성 (템플릿 선택)
- 미션 생성 (커스텀)
- 미션 상세
- 미션 진행 현황

#### 2.2.5 금융 화면
- 오픈뱅킹 연동
- 계좌 목록
- 계좌 상세
- 거래 내역
- 자산 대시보드

#### 2.2.6 프로필 화면
- 프로필 설정
- 비밀번호 변경
- 회원 탈퇴

#### 2.2.7 보상 화면
- 뱃지 목록

### 2.3 공통 컴포넌트

#### 2.3.1 하단 네비게이션 바
**위치**: 화면 하단 고정

**구성 요소**:
- 홈 (대시보드) 아이콘
- 미션 아이콘
- 자산 아이콘
- 프로필 아이콘

**동작**:
- 현재 화면 하이라이트
- 탭 시 해당 화면으로 이동

#### 2.3.2 상단 헤더
**위치**: 화면 상단

**구성 요소**:
- 화면 제목 (또는 로고)
- 알림 아이콘 (선택적)
- 설정/메뉴 아이콘 (선택적)

#### 2.3.3 카드 컴포넌트
**스타일**:
- 둥근 모서리
- 그림자 효과
- 충분한 패딩
- 클릭 가능 시 호버 효과

#### 2.3.4 버튼
**타입**:
- Primary: 주요 액션 (예: 미션 시작, 저장)
- Secondary: 보조 액션 (예: 취소, 뒤로)
- Text: 텍스트 링크 형태

**크기**:
- Large: 주요 액션 (높이 48px 이상)
- Medium: 일반 액션 (높이 40px)
- Small: 보조 액션 (높이 32px)

#### 2.3.5 진행률 표시
**형태**:
- 프로그레스 바
- 원형 프로그레스 (선택적)
- 퍼센트 표시

### 2.4 화면 상세 설계

#### 2.4.1 인증 화면

##### 2.4.1.1 로그인 화면

**레이아웃**:
```
┌─────────────────────────┐
│      [로고/브랜드]        │
│                         │
│   Nestack에 오신 것을    │
│      환영합니다          │
│                         │
│  ┌───────────────────┐ │
│  │ 이메일             │ │
│  └───────────────────┘ │
│                         │
│  ┌───────────────────┐ │
│  │ 비밀번호           │ │
│  └───────────────────┘ │
│                         │
│  [ ] 자동 로그인        │
│                         │
│  ┌───────────────────┐ │
│  │      로그인        │ │
│  └───────────────────┘ │
│                         │
│  ┌───────────────────┐ │
│  │  Google로 로그인   │ │
│  └───────────────────┘ │
│                         │
│    비밀번호 찾기        │
│    회원가입             │
└─────────────────────────┘
```

**구성 요소**:
- 로고/브랜드명
- 환영 메시지
- 이메일 입력 필드
- 비밀번호 입력 필드 (비밀번호 표시/숨김 토글)
- 자동 로그인 체크박스
- 로그인 버튼 (Primary, Large)
- Google 소셜 로그인 버튼
- 비밀번호 찾기 링크
- 회원가입 링크

**상태**:
- 입력 오류: 필드 하단에 오류 메시지 표시
- 로딩: 버튼에 로딩 스피너 표시
- 성공: 다음 화면으로 이동

- 오류 메시지 즉시 표시

  - 금액
  - 날짜

- 하단 네비게이션 바

**상태**:
- 가족 그룹 미연결: 가족 연결 안내 카드 표시
- 계좌 미연동: 계좌 연동 안내 카드 표시
- 미션 없음: 미션 생성 유도 메시지

### 2.5 사용자 플로우

#### 2.5.1 신규 사용자 온보딩 플로우
1. 로그인 화면 → 회원가입
2. 회원가입 완료 → 이메일 인증
3. 이메일 인증 완료 → 대시보드
4. 대시보드 → 가족 그룹 생성 안내
5. 가족 그룹 생성 → 초대 코드 발급
6. 배우자 초대 → 계좌 연동 안내
7. 계좌 연동 → 미션 생성 안내

#### 2.5.2 미션 생성 플로우
1. 미션 목록 → 미션 생성 버튼
2. 템플릿 선택 또는 직접 만들기
3. 미션 상세 설정 (목표 금액, 기한)
4. 데이터 공유 범위 확인 및 동의
5. 미션 시작 → 미션 목록에 추가

#### 2.5.3 거래-미션 연결 플로우
1. 거래 내역 → 거래 선택
2. 미션 연결 버튼
3. 연결할 미션 선택
4. 연결 완료 → 미션 진행률 업데이트

- 전체 화면 로딩 시 스켈레톤 UI 또는 로딩 인디케이터

## 3. 어드민 화면

### 3.1 디자인 원칙

#### 3.1.1 디자인 컨셉
- **데스크톱 중심**: 관리자가 효율적으로 작업할 수 있는 데스크톱 인터페이스
- **데이터 중심**: 대량의 데이터를 빠르게 조회하고 관리
- **기능 접근성**: 주요 기능을 빠르게 접근할 수 있는 사이드바 네비게이션
- **정보 밀도**: 한 화면에 많은 정보를 효율적으로 표시

#### 3.1.2 레이아웃 원칙

#### 3.1.3 색상 및 타이포그래피

- **상태 색상**: 활성/비활성, 성공/실패 등 상태를 명확히 구분

### 3.5 사용자 플로우

#### 3.5.1 어드민 로그인 플로우
1. 어드민 로그인 화면
2. 이메일/비밀번호 입력
3. 로그인 성공 → 대시보드

#### 3.5.2 미션 템플릿 생성 플로우
1. 미션 템플릿 목록 → 미션 템플릿 생성 버튼
2. 미션 템플릿 생성 화면
3. 정보 입력 (미션명, 카테고리, 설명 등)
4. 저장 → 미션 템플릿 목록에 추가

#### 3.5.3 뱃지 수동 발급 플로우
1. 뱃지 발급 이력 → 수동 발급 버튼
2. 사용자 선택
3. 뱃지 선택
4. 발급 완료 → 이력에 추가
